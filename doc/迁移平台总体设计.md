## 1. 定位

本平台是面向异构数据库的端到端数据迁移系统，覆盖结构迁移、全量迁移、增量同步、数据校验与业务切换，并提供 AI 智能辅助（映射建议与异常诊断）。

## 2. 能力范围

| 能力 | 说明 |
|------|------|
| 结构迁移 | Schema 采集、类型映射、目标端对象落地（DBCat） |
| 全量迁移 | 存量数据切片并发搬运（FullImport） |
| 增量同步 | 变更日志拉取与应用（Store / IncrSync） |
| 数据校验 | 源/目标全字段对比与订正（FullVerification） |
| 智能辅助 | 映射建议、异常诊断、差异解释（AIEngine） |
| 运维管控 | 任务编排、监控告警、组件高可用（Supervisor） |

## 3. 逻辑结构

端到端形态为：**源数据库 → 迁移平台 → 目标数据库 / 消息队列**。

平台内部包括：集中管控、基础服务（集群/资源/高可用/元数据）、结构与全量、增量日志管道（读取 ↔ 写入）、AI 智能辅助。

模块按三层组织：

1. **服务接入层**：传输项目管理、数据源管理、运维监控、告警设置  
2. **流程编排层**：对象迁移、数据迁移、数据同步、数据校验、链路切换  
3. **组件层**：DBCat、Store、FullImport、IncrSync、FullVerification、AIEngine，由 Supervisor 统一监控  

## 4. 标准迁移流程

```text
创建任务 → 预检查 → 结构迁移 → 全量迁移 → 增量同步 → 数据校验 → 业务切换 → 完成
```

全量启动时记录增量起始位点；增量追平后触发校验；校验通过后再切换业务流量。结构映射与差异分析可调用 AIEngine 辅助。
