## 1. 定位

本平台是面向异构数据库的端到端数据迁移系统，覆盖结构迁移、全量迁移、增量同步、数据校验与业务切换，提供统一控制台与 OpenAPI。

## 2. 能力范围

| 能力 | 说明 |
|------|------|
| 结构迁移 | Schema 采集、类型映射、目标端对象落地（DBCat） |
| 全量迁移 | 存量数据切片并发搬运（FullImport / SyncWorker） |
| 增量同步 | 变更日志拉取与应用（Store / IncrSync） |
| 数据校验 | 源/目标全字段对比与订正（FullVerification） |
| 运维管控 | 任务编排、监控告警、组件高可用 |

## 3. 逻辑分层

自上而下分为四层：

1. **服务接入层**：控制台、API 网关、认证鉴权  
2. **流程编排层**：任务编排、工作流、元数据、数据源管理  
3. **执行引擎层**：DBCat、FullImport、Store、IncrSync、FullVerification  
4. **基础设施层**：调度、资源池、HA、监控、Supervisor  

控制平面负责任务决策与状态；数据平面负责实际读写，二者通过调度器解耦。

## 4. 标准迁移流程

```text
创建任务 → 预检查 → 结构迁移 → 全量迁移 → 增量同步 → 数据校验 → 业务切换 → 完成
```

全量启动时记录增量起始位点，增量追平后触发校验，校验通过后再切换业务流量。
